## product-doc-lifecycle skill 增加 `08-user-guide` 文档

### 背景
用户要求增加"产品使用说明"文档，在项目交付时提供给最终用户。

### 决策结论
- **文档名称**：`08-user-guide.md`（kebab-case，与现有 01-07 编号连续）
- **定位**：面向最终用户（非开发者），与 `05-feature-guide`（面向开发者）明确区分
- **必要性**：有终端用户界面的项目（Web UI / 移动端 / 桌面端）建议建；纯 API / 库项目不建
- **触发时机**：S5 部署阶段（产品交付前）
- **复杂度**：简单项目可选，中等/复杂项目建议建

### 关键设计决策
- **为什么不是必建**：纯 API 项目、库/框架项目没有"终端用户"，强行建是浪费
- **为什么和 05 分开**：05 面向开发者（含配置项、技术细节），08 面向最终用户（纯操作步骤、非技术语言）
- **触发在 S5 而不是 S6**：部署后就要交付给用户，使用说明必须在交付前就绪
